# PROJECT_STRUCTURE_AND_ROADMAP.md

# 智能问答知识库项目结构与开发路线

本文档用于说明本项目的推荐目录结构、模块职责、开发顺序和阶段性验收标准。

本项目目标是构建一个面向工程学习、项目知识理解和智能问答的 AI Engineering Assistant。

---

## 1. 项目最终目标

系统最终应支持：

```txt
基础对话
流式输出
会话持久化
Agent Tool Use：Web Search 外部信息检索
工具调用过程可视化
文档上传
Docs RAG 检索
Citation 引用展示
Wiki 知识管理
Code Grep 代码检索
Router 证据路由
```

但开发时不要一次性实现全部功能。

推荐路线：

```txt
对话底座 -> Agent 工具调用 -> Agent Loop 可视化 -> 会话持久化 -> 知识库 -> 多来源检索
```

## 3. 核心目录说明

### 3.1 `src/app`

负责 Next.js 页面和 API 路由。

放：

```txt
页面
布局
API route 入口
```

不放：

```txt
复杂业务逻辑
数据库细节
Agent Runtime
工具执行逻辑
RAG 细节
```

### 3.2 `src/components`

负责前端 UI 组件。

组件应按业务模块划分：

```txt
chat       聊天相关
agent      Agent 活动面板
tools      工具调用展示
sources    引用来源展示
documents  文档管理
wiki       Wiki 管理
ui         通用基础组件
```

### 3.3 `src/lib`

负责核心业务逻辑。

```txt
chat       聊天流程
llm        模型调用
agent      Agent Runtime
tools      工具系统
db         数据库
rag        文档索引和向量检索
retrieval  多来源检索
storage    文件存储
code       代码搜索
queue      异步任务
types      类型
utils      工具函数
```

## 7. 流式事件设计

后端向前端返回结构化事件。

### 第一版事件

```ts
export type BasicStreamEvent =
  | {
      type: "message_delta";
      content: string;
    }
  | {
      type: "done";
    }
  | {
      type: "error";
      message: string;
    };
```

### Agent 阶段事件

```ts
export type StreamEvent =
  | {
      type: "message_delta";
      content: string;
    }
  | {
      type: "agent_step";
      stepId: string;
      title: string;
      status: "running" | "done" | "error";
    }
  | {
      type: "tool_call";
      toolCall: ToolCall;
    }
  | {
      type: "tool_result";
      result: ToolResult;
    }
  | {
      type: "sources";
      sources: CitationSource[];
    }
  | {
      type: "done";
    }
  | {
      type: "error";
      message: string;
    };
```

## 8. 类型设计

### Message

```ts
export type MessageRole = "user" | "assistant" | "system" | "tool";

export type ChatMessage = {
  id: string;
  conversationId: string;
  role: MessageRole;
  content: string;
  toolCalls?: ToolCall[];
  citations?: CitationSource[];
  createdAt: string;
};
```

### ToolCall

```ts
export type ToolCall = {
  id: string;
  name: string;
  arguments: Record<string, unknown>;
};
```

### ToolResult

```ts
export type ToolResult = {
  toolCallId: string;
  name: string;
  success: boolean;
  content: string;
  metadata?: Record<string, unknown>;
};
```

### CitationSource

```ts
export type SourceType = "doc" | "wiki" | "code" | "web" | "tool";

export type CitationSource = {
  id: string;
  type: SourceType;
  title: string;
  locator: string;
  uri?: string;
  snippet: string;
  score?: number;
  metadata?: Record<string, unknown>;
};
```

# Phase 3：/api/chat 基础接口

## 目标

接入最小可用的真实后端对话链路。

```txt
前端使用 useChat
后端使用 LangChain.js
/api/chat 返回 AI SDK 兼容的流式响应
先只实现最简单的文本对话
```

说明：

```txt
本阶段提前合并原 Phase 4 的一部分能力。
原因是本项目后续明确要承接 Agent / RAG / Tool Use，
因此从现在开始统一后端主链路，避免后续再次迁移模型调用层。
```

## 任务

```txt
实现 POST /api/chat
接入真实模型的最小对话能力
前端从本地 mock 流切换为 useChat
后端使用 LangChain.js 产出流式文本
route.ts 只保留 API 入口
聊天逻辑下沉到 lib/chat
模型创建与流式调用下沉到 lib/llm
先只支持 user / assistant 纯文本消息
为后续 agent_step / tool_call / tool_result / sources 预留结构位
```

## 当前方案

```txt
前端：
- 使用 @ai-sdk/react 的 useChat 管理消息、状态、发送和中断
- 保留当前自定义 Chat UI、Virtuoso、Sidebar、ToolPanel 布局
- 本阶段先只渲染纯文本 assistant 回复

后端：
- 使用 Next.js App Router 的 /api/chat route handler
- 使用 LangChain.js 作为后端模型调用主链路
- 返回 AI SDK 兼容的流式响应，确保 useChat 可以直接消费
- 当前不接 Agent、不接 RAG、不接 Tools，只保留演进边界

分层：
- app/api/chat/route.ts：API 入口
- src/lib/chat/handle-chat-request.ts：聊天请求编排
- src/lib/llm/create-chat-model.ts：创建 LangChain chat model
- src/lib/llm/stream-chat.ts：执行流式输出
- src/lib/types/chat.ts：聊天请求和消息类型
```

## 验收标准

```txt
前端消息来自 /api/chat
前后端已经打通
真实模型回复可以流式显示
前端已经切换为 useChat
接口错误时前端能展示错误
当前实现未把 Agent / RAG / Tool Use 逻辑写死在 route.ts
后续可以在 LangChain 主链路上继续扩展 Agent / RAG / Tool Use
```

